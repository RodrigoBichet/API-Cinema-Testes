# Template do Bug Report

```markdown
## [Título do Bug]

**Descrição Resumida:**
_Uma breve descrição do problema encontrado._

**Gravidade:**
_Selecione a gravidade do bug (Crítico, Alto, Médio, Baixo)._

**Prioridade:**
_Indique a prioridade para correção (Alta, Média, Baixa)._

**Classificação:**
_Indique a classificação do bug (DEV, ANÁLISE, MASSA ou AMBIENTE)._
_DEV: O bug pode ter ocorrido devido a uma falha no desenvolvimento da demanda._
_ANÁLISE: Especificações funcionais e técnicas elaboradas erroneamente ou que não suprem a necessidade do cliente._
_MASSA: Inconsistência nos dados utilizados para testes._
_AMBIENTE: Falha no ambiente que está sendo utilizado para testes. Nesse caso pode ser necessário entrar em contato com a fornecedora do produto ou administrador do sistema._

**Passos para Reproduzir:**

1. [Passo 1: Descreva detalhadamente a ação realizada]
2. [Passo 2: Continue descrevendo os passos até que o bug ocorra]
3. [Passo 3: Último passo antes de o problema acontecer]

**Resultado Esperado:**
_Explique o comportamento esperado da aplicação._

**Resultado Obtido:**
_Descreva o comportamento real observado quando o bug ocorreu._

**Ambiente de Teste:**

-   **Sistema Operacional:** [ex.: Windows 10, Ubuntu 20.04]
-   **Navegador (se aplicável):** [ex.: Chrome 91.0, Firefox 89.0]
-   **Versão da Aplicação:** [ex.: 1.0.0]
-   **Configuração de Hardware (se aplicável):** [ex.: 8 GB RAM, Intel Core i5]
-   **Resolução de Tela (se aplicável):** [ex.: 1920x1080]
-   **Outros Detalhes Relevantes:** [ex.: plugins ativos no navegador, VPN, rede corporativa]

**Logs ou Mensagens de Erro:**
_Se houver, inclua logs de erro, mensagens de console ou capturas de tela do erro._

**Anexos:**

-   **Capturas de Tela:** [_inserir link ou imagem_]
-   **Vídeo (se aplicável):** [_inserir link ou arquivo de vídeo_]
-   **Logs do Sistema:** [_inserir logs_]

**Possível Solução Proposta (Opcional):**
_Se você tiver uma ideia de como o problema pode ser resolvido, descreva-a aqui._

**Notas Adicionais:**
_Qualquer outra informação que possa ser útil para o time de desenvolvimento entender melhor o problema._
```
